# Dragon Ball API - Route Documentation

This document provides a visual representation and explanation of all the available routes in the Dragon Ball API. Each image corresponds to a specific route, followed by a brief explanation of its functionality.

---

## 1. Get Random Question

![Random Question Endpoint](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_1_2024-09-21_17-59-47.jpg?raw=true)

This route returns a random question and answer related to Dragon Ball from the database.

---

## 2. Get by ID

![Get by ID Endpoint](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_10_2024-09-21_17-59-47.jpg?raw=true)

This route retrieves data by a specific ID. Pass the ID in the URL to get the corresponding data from the database.

---

## 3. Add Data to API

![Add Data Endpoint](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_3_2024-09-21_17-59-47.jpg?raw=true)

Use this route to add new data to the Dragon Ball API. The request is a POST request, and the database is updated accordingly.

---

## 4. Update Data (PUT)

![PUT Request](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_2_2024-09-21_17-59-47.jpg?raw=true)

This route is used to fully update a record in the API. The `PUT` request allows you to replace the existing data for a specific ID with new information.

![PUT Request in Database](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_9_2024-09-21_17-59-47.jpg?raw=true)

The database is updated after the PUT request is made.

---

## 5. Partially Update Data (PATCH)

![PATCH Request](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_17_2024-09-21_17-59-47.jpg?raw=true)

This route allows for partial updates to a record. A PATCH request modifies only the specific fields you provide.

![PATCH Request in Database](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_16_2024-09-21_17-59-47.jpg?raw=true)

The database reflects the partial updates after the PATCH request is executed.

---

## 6. Get Admin Token

![Get Token](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_5_2024-09-21_17-59-47.jpg?raw=true)

To authenticate as an admin, use this route to obtain a token. You must provide valid admin credentials to receive the token.

---

## 7. Delete One Record (Authenticated)

![Delete One Record](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_20_2024-09-21_17-59-47.jpg?raw=true)

This authenticated route allows you to delete a specific record from the database.

![Database after Deletion](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_6_2024-09-21_17-59-47.jpg?raw=true)

The database is updated after the deletion is performed.

---

## 8. Delete All Records (Authenticated)

![Delete All Records](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_19_2024-09-21_17-59-47.jpg?raw=true)

This route allows an authenticated admin to delete all records in the database.

![Database Re-initialized After Deletion](https://github.com/MishraShardendu22/Dragon-Ball-Api/blob/main/DB%20-%20API/photo_8_2024-09-21_17-59-47.jpg?raw=true)

After the deletion of all records, the database is automatically reinitialized with default data.
